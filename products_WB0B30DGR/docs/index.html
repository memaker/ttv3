Features

Ace is a lightweight and feature rich admin template which is clean and easy to use.
Current release comes with the following features:

  • 4 different theme colors
  • Responsive design based on Bootstrap 2.3.1
  • Custom elements and plugins including:
    • Pure CSS3 checkbox, radio and On/Off switch elements (not available in IE8)
    • File input control
    • Simple color picker
    • Customizable widgets in different colors and styles
  • Bootstrap, jQuery UI and third party plugins and elements including:
    • Charts
    • Dynamic tables
    • Slider
    • Spinner
    • Wizard
    • Form validation
    • Color picker
    • Date picker
    • DateRange picker
    • Time picker
    • Colorbox for gallery
    • Full calendar
    • Chosen selects
    • Textarea input limiter
    • Textarea auto resize
    • Masked input
    • Growl like notifications
    • Imageless spinner
Note Stylings of most third party elements have been modified to replace external images and icons with CSS and FontAwesome.

This has benefits such as better view on Retina displays as almost no pixel graphics are used and it also means fewer network file requests resulting in faster page load time.

Theme Colors

Currently there are 3 different theme colors for Ace, other than the default one.
You can choose each one from the Settings button on right side of each page just below the search box.
However when you change a setting no data will be saved and by refreshing the page you will see the default theme again.
In your application you can set the theme on body tag in a static way: 
<body class="skin-1">
<body class="skin-2">
<body class="skin-3">
or dynamically in your server side script: 
<body class="<?php echo $_COOKIE['skin']; ?>">

Also you may notice when you choose a skin, the top user menu buttons change their color.
This is not dependent on the skin but the color class of the buttons are changed via Javascript.
You can hardcode them inside your application to any desired color as specified in next section.

The Empty File

Let's continue with general structure of empty.html file as an example.

The top navigation is as followes: 
<div class="navbar navbar-inverse">
  <div class="navbar-inner">
   <div class="container-fluid">

     <a class="brand" href="#"> Logo goes here </a>

     <ul class="nav ace-nav pull-right">
       The top user menu goes here
     </ul>

   </div>
  </div>
</div>

Use the top user notifications menu .ace-nav like a normal menu: 

<li class="green">
   <a href="#" class="dropdown-toggle" data-toggle="dropdown">
      <i class="icon-tasks"></i>
      <span class="badge">4</span>
   </a>
	
   <ul class="pull-right dropdown-navbar dropdown-menu">
       <!-- DROP DOWN MENU ITEMS HERE -->
   </ul>
</li>
This li element can have any of the following classes:
.grey .purple .green .light-blue .red .light-green .light-purple .light-orange .dark .light-blue2 .white-pink
.no-border .margin-1 .margin-2

The final element also has the class .user-profile
Please check the html source for this item and include the classes as used for better responsiveness.

The #main-container element contains everything else (sidebar and content) and starts with : 

<a href="#" id="menu-toggler"><span></span></a>
which is the button to toggle sidebar menu in mobile and tablet devices.

Next comes the #sidebar element: 

<div id="sidebar">
</div>
It begins with shortcut buttons on top which you can delete or replace with search box element #nav-search
Next is the menu itself and then the sidebar collapse button, which helps minimize the menu, for more content space.

Note When you minimize the menu, nothing is saved and when you refresh the page the full menu is displayed again. To save the state in your application you may use cookies (support coming in next updates) or in your server side application: 
<div id="sidebar" class="<?php if(isset($_COOKIE['menu_state'])) 
	                               AND $_COOKIE['menu_state'] == 'min') 
		              echo 'menu-min'; ?>">
</div>

Then is #main-content which begins with optional #breadcrumbs which has breadcrumbs and the #nav-search followed by "#page-content" element containing the page contents.

After that it's the optional settings container #ace-settings-container.

That was a brief overview of the page structure and you can empty.html and blank.html for more details.

UI Elements

Tabs

Tab content .tab-content may have different paddings based on different needs (for examples tables in tabs with no padding):
.no-padding .padding-4 .padding-8 .padding-24

You can also use tabs inside widget header which is shown on widgets section.

Progress Bar

There are two additional classes .progress-small and .progress-mini for smaller sized progress bars.
You may also use data-percent attribute to display the percentage value on center of the .progress element. 
<div class="progress progress-mini" data-percent="62%"> <div class="bar" style="width:62%"></div> </div>

The are also circular progress bars (using easy pie chart) for which you can refer to the documentation or examples in html files.

Tooltips and Popovers

There are four more classes related to tooltip and popover coloring:
.tooltip-error .tooltip-info .tooltip-warning .tooltip-success
Apply these classes to the element for which you are going to invoke the tooltip function.
It applies to both popovers and tooltips.
Popovers can also be without title .no-title 
<div class="tooltip-error" title="Tooltip Text!" data-placement="top">
    Show error tooltip on hover!
</div>

Dropdowns

Dropdown menus have some extra classes.
For different menu colors on hover and active states use:
.dropdown-default .dropdown-info .dropdown-purple .dropdown-success .dropdown-warning .dropdown-danger .dropdown-inverse .dropdown-pink .dropdown-yellow .dropdown-grey .dropdown-light .dropdown-lighter
Use .dropdown-caret to show the caret.
Use .dropdown-close and .dropdown-closer to show the menu a bit closer to the invoker element. 
<ul class="dropdown-menu dropdown-purple dropdown-closer dropdown-caret">
</ul>

Form Elements

CSS3 Checkbox and Radio

Ace comes with Pure CSS3 styled checkbox without any javascript. Switches are also animated on supported browsers such as Firefox and Latest Chrome. It also fall back to default browser style on IE8.

Samples for checkboxes and radio: 

<input type="checkbox" id="checkbox1" /> <label class="lbl" for="checkbox1"> check me</label>

<label> <input type="checkbox" /> <span class="lbl"> check me</span> </label>

Note that you must include an element with .lbl class right after the checkbox or radio. It may or may not contain text.
Checkboxes have a second style which you should add .ace-checkbox-2 class to the input element. 
<input type="checkbox" id="swith1" class="ace-switch" /> <label class="lbl" for="swith1"></label>

<label> <input type="checkbox" class="ace-switch ace-switch-7" /> <span class="lbl"> check me</span> </label>

Currently switches come in 7 styles: .ace-switch , .ace-switch.ace-switch-2, ... .ace-switch-.ace-switch-7


Custom File Inputs

File inputs come in 2 different formats, small and large with drag & drop ability

<input type="file" id="file-input-1" />
<script>
$('#file-input-1').ace_file_input({
   no_file:'No File ...',
   btn_choose:'Choose',
   btn_change:'Change'
}).on('change', function(){
	var files = $(this).data('ace_input_files');
	//upload files, etc ...
};
</script>

<input type="file" multiple />
<script>
$('input[type=file][multiple]').ace_file_input({
   style:'well',/** for making it like the larger file input in the example */
   btn_choose:'Drop files here or click to choose',
   btn_change:null,
   no_icon:'icon-cloud-upload',
   droppable:true,
   thumbnail:'large',
   /** before_change:function(files, dropped) {
      return false;//to cancel
      return true or a modified file array
   },
   before_remove:function() {
      //called right after clicking the red remove button
      return false;//to cancel reseting the input
   }
   */
});
</script>

After the files have been selected, jQuery data fields ace_input_files and ace_input_method are set for the the file input element.

You can retrieve them like this: 

var files = $('file input element').data('ace_input_files');// which is an Array or FileList containing the selected files.
var method = $('file input element').data('ace_input_method');// which is either "drop" or "select" depending on how the user selected files.

But before this data is set, before_change function is called if it is defined.
It has two arguments, a FileList and a boolean which tells us whether files have been selected or dropped.

You can check the file list and remove some if not desired before being saved to field data.
If you return false nothing will be saved/changed, if you return true all files will be saved and if you return a (modified) file array it will be saved instead.

Hint For an example see the form elements page.

However please note that ace_input_files is only useful for javascript handling and uploading of files.
The file input field's value cannot be changed to reflect your changes because of the browser's security measures. (it can only be reset)
If you submit the form like traditional forms and files are sent as usual, all files are sent to server for further processing.

Currently the "Ace File Input" element may be good in most situtations but more flexibility and features will be added in next updates.
